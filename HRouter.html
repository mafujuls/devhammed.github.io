HRouter.js Javascript Routing Library

HRouter.js is a Simple and Lightweight Javascript Routing Library suitable and designed for Single-page Web Applications and Mobile HTML5 Applications.

How to Use & Examples

HRouter.js doesnt require setup or installation, Download it HERE and Include it in your HTML5 Project then you Register/Add Routes using HRoute function, this function takes 3 Arguments.

Path : The Route URL e.g /about (required)

.

Page ID : The ID attribute value of the Page Section Tag (required)

Callback : The function to automatically executed when the page load, you can use this to change Page Title, Header, Footer or Other parts of the page you dont want it to be same with other pages. (optional)

e.g

<script>
// home page
HRoute('/', 'home', function () {
document.title = "Home page";
});
// about page
HRoute('/about', 'about', function () {
document.title = "About page";
});
</script>

Note: HRouter.js would redirect to 404 Error Page if no Route was registered/found for the current URL.

Page Layout & Structure

MAIN tag is used as the Main Contents container, it must have ID attribute value of 'hr-app' and should be left blank as HRouter.js would use it as Page Renderer.

HEADER tag should be used as the container for the Header Contents, it must have ID Attribute value of 'hr-header'.

FOOTER tag should be used as the Container for Footer Contents, it must have ID attribute value of 'hr-footer'.

SECTION tag should be used to create Pages in your Single-page Application, each Page SECTION tag must have the CLASS attribute value of 'hr-page' and also have ID attribute to identify the Page when using HRoute() for Routing or when Linking to the Page. To link to the Page, use Syntax:-
<a href="#/PAGE_ID">Link Text</a>
(Replace PAGE_ID with the Page Section Tag ID) and to Route a URL to the Page use Syntax:

<script>
HRoute('/PAGE_URL', 'PAGE_ID', function () {
document.title = 'PAGE_TITLE';
});
</script>

(Replace PAGE_URL with the Route URL, PAGE_ID with Page Section Tag ID and PAGE_TITLE with the Page Title.

NAV tag should be used as the container for group of Navigation Links.

ASIDE tag should be used as the container for Sidebar Contents.

HLOAD tag should be used to Load Content from a Template File, It must have 'data-url' attribute with the value of the Template URL. e.g to Load contactform.html content into the Contact Us Section Tag.

<section id="contact-us" class="hr-page">
<h1>Contact us</h1>
<p>Use below form to contact us.</p>
<hload data-url="contactform.html"></hload>
</section>

Above tags are should be used only for the purpose specified above, Other HTML Tags (DIV, TABLE, SCRIPT, STYLE, INPUT, BUTTON, CANVAS, SVG, SPAN, EM, B etc) can be used normally.

Hints and Tips

1. Main Application/Website Contents can be Styled by Selecting 'hr-app' by ID in your CSS Definitions e.g

#hr-app {
/* Styles for Main Contents */
}


2. Header Content can be Styled by Selecting 'hr-header' by ID and Footer Content by Selecting 'hr-footer' by ID in your CSS Definitions e.g

#hr-header {
/* Styles for Header Contents */
}

#hr-footer {
/* Styles for Footer Contents */
}


3. You can also use GET(name) function to capture Query Strings Value in URL e.g

<script>
// if url is: http://www.example.com/post.html?id=4&action=view
var id = GET('id');
var action = GET('action');
// Do something with 'id' & 'action' value :-)
</script>


4. You also have full access to all Web APIs like Web Storage, Canvas, SVG, FileReader, FileWriter, Offline App Cache, Geolocation, Battery API, Vibrate API, AJAX/XMLHttpRequest, IndexedDB, Web RTC, Web Workers, Web Socket, Drag and Drop, Web SQL etc.
